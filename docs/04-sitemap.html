2 ページ一覧

Webサイト ページ一覧 必須
No.ページ名ページの目的主要コンテンツ
1トップページサービスの魅力を伝え、問い合わせや資料請求につなげるキャッチコピー、サービス概要、主な機能、導入メリット、CTA
2サービス概要JINJIROがどのような課題を解決するサービスかを説明するサービス説明、解決できる課題、対象ユーザー、導入メリット
3機能紹介提供機能を具体的に紹介し、利用イメージを持ってもらう人事データ一元管理、ダッシュボード分析、勤怠管理、評価管理、給与管理
4導入事例導入後の活用イメージや効果を伝えて信頼感を高める企業事例、活用シーン、導入効果、担当者コメント
5料金プラン料金体系を明確にし、検討しやすくする料金プラン一覧、プランごとの機能比較、導入条件
6資料ダウンロード詳細情報を提供し、見込み顧客のリード獲得につなげるダウンロードフォーム、資料概要、掲載内容
7お知らせ最新情報やアップデート情報を発信する新着情報、機能追加情報、セミナー案内
8お問い合わせ導入相談や質問を受け付ける問い合わせフォーム、相談内容入力欄、連絡先情報
Webアプリ 画面一覧 必須
No.画面名画面の目的主要機能
1ログイン画面ユーザー認証メールアドレス入力、パスワード入力、ログイン処理
2ダッシュボード画面人事データの概要を確認する人事データサマリー、勤怠状況、評価状況
3社員一覧画面社員情報を一覧表示する検索、ソート、ページネーション
4社員詳細画面社員の詳細情報を確認する社員プロフィール表示、評価履歴表示
5社員登録画面新しい社員情報を登録する入力フォーム、バリデーション、登録処理
6社員編集画面社員情報を更新する編集フォーム、更新処理
7勤怠一覧画面社員の勤怠データを確認する勤怠一覧表示、検索、フィルター
8評価管理画面社員の評価情報を管理する評価一覧表示、評価登録、評価編集
9レポート画面人事データを分析するグラフ表示、レポート確認
10アカウント設定画面ユーザー情報を管理するプロフィール編集、パスワード変更、ログアウト

3 ユーザーフロー

メインシナリオ 必須
ペルソナがサイトに訪問してから目的を達成するまでの動線（メインの行動パターン）

検索エンジンで「人事管理システム」「HRシステム」などを検索
→ JINJIROのトップページを訪問
→ サービス概要を確認
→ 機能紹介ページで人事データ管理・ダッシュボード機能を確認
→ 導入事例ページで他社の活用事例を確認
→ 料金プランページで導入コストを確認
→ 資料ダウンロードページでサービス資料を取得
→ 社内で共有し導入検討
→ 問い合わせフォームからデモ・相談を依頼

サブシナリオ 任意
その他の行動パターン（1〜2パターン）

同僚や人事担当者からHRシステムの情報を聞く
→ SNS（LinkedIn・X）でHRツールの情報を検索
→ HRツール比較サイトの記事を閲覧
→ JINJIROのサービス紹介ページを訪問
→ 機能紹介ページで人事データ管理機能を確認
→ 導入事例ページで他社の活用事例を確認
→ 他のHRシステムと機能や料金を比較
→ 無料デモや資料ダウンロードを検討
→ 問い合わせページからサービス詳細を確認
